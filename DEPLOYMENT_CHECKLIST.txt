## Phase 3: Frontend Deployment üåê

### Prepare for Deployment
- [ ] Choose hosting: Vercel (recommended) or Netlify
- [ ] Push code to GitHub
- [ ] Update all placeholder URLs in code
- [ ] Prepare images:
  - [ ] Embed image (3:2 ratio, 600x400px min)
  - [ ] Splash screen (200x200px)
  - [ ] Icon/Logo (square)
  - [ ] Screenshots for Mini App store

### Upload Images
- [ ] Upload to IPFS (Pinata or NFT.Storage) or CDN
- [ ] Update image URLs in:
  - [ ] `frontend/public/.well-known/farcaster.json`
  - [ ] `frontend/app/layout.tsx` (embed metadata)

### Deploy Frontend
- [ ] Connect GitHub repo to Vercel/Netlify
- [ ] Set environment variables:
  - NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID
  - NEXT_PUBLIC_ENABLE_TESTNETS (false for prod)
- [ ] Deploy!
- [ ] Get production URL
- [ ] Test on production URL

## Phase 4: Farcaster Mini App Setup üì±

### Update farcaster.json
- [ ] Replace all `https://your-domain.com` with actual domain
- [ ] Update `baseBuilder.ownerAddress` with your Base Account address
- [ ] Update all image URLs
- [ ] Update `homeUrl` to your production URL
- [ ] Ensure file is accessible at `/.well-known/farcaster.json`

### Generate Account Association
- [ ] Visit https://base.build
- [ ] Use Account Association tool
- [ ] Enter your domain (e.g., basematch.vercel.app)
- [ ] Click "Submit" then "Verify"
- [ ] Copy generated `accountAssociation` fields
- [ ] Paste into `farcaster.json`
- [ ] Deploy updated farcaster.json

### Test Mini App
- [ ] Visit https://base.build/preview
- [ ] Enter your app URL
- [ ] Verify embeds display correctly
- [ ] Test launch button works
- [ ] Check all metadata appears
- [ ] Test on mobile (Warpcast app)

### Publish to Base
- [ ] Open Base app (Warpcast)
- [ ] Create a post with your app URL
- [ ] Verify rich embed appears
- [ ] Test "Find Your Match" button
- [ ] Share with friends for testing
